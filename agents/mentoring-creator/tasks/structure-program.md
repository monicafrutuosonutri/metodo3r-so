## Executive Summary

Desenhar a estrutura macro do programa: fases, timeline, sessoes, entregaveis, checkpoints. Adaptado pela modalidade. Executa CONTRA o PRD — toda decisao de estrutura deve ser coerente com o que foi prometido.

**Gate:** QG-MC-004 — Estrutura Validada

## Steps

### Step 1: Contextualizar

- Reler PRD (modalidade, parametros, metodologia)
- Reler metodologia mapeada na Fase 2
- Definir como a metodologia se traduz em fases do programa

### Step 2: Co-Desenvolver Estrutura

Usar ciclo fractal:
1. Trazer proposta baseada na metodologia + pesquisa
2. Expert reage
3. Provocar: "Faz sentido essa sequencia?" "Falta alguma fase?"
4. Ajustar
5. Repetir ate fechar

### Step 3: Template por Modalidade

**INDIVIDUAL:**
- Fases do programa (ex: diagnostico → estrategia → implementacao → revisao → consolidacao)
- Sessoes 1:1: quantidade, frequencia, duracao
- Assessment de entrada
- Plano personalizado
- Canais de suporte async
- Checkpoints de progresso

**GRUPO:**
- Formato: cohort ou ongoing
- Tamanho de turma
- Calendario de encontros
- Tipos de sessao por encontro
- Comunidade (plataforma, regras)
- Eventos especiais (se houver)

**CONSULTORIA:**
- Fases do engagement
- Deliverables por fase
- Nivel DWY/DFY por fase
- VIP Days ou sprints
- SLA de entrega

### Step 4: Validar contra PRD

Verificar:
- Duracao bate com PRD?
- Frequencia bate?
- Entregaveis prometidos estao cobertos?
- Parametros respeitados?

### Step 5: Apresentar e Aprovar

Mostrar estrutura completa em formato legivel. Debate ate aprovacao.

## Quality Gate QG-MC-004

Protocolo de 5 passos:

1. **APRESENTAR** estrutura completa em formato legivel
2. **PERGUNTAR** "Essa estrutura faz sentido? Falta alguma fase? Mudaria algo?"
3. **DEBATE** — ajustar ate satisfeito (sem limite de rounds)
4. **APROVACAO** — usuario diz "fechou"
5. **HANDOFF** — passar pra Fase 4 (design-sessions) com estrutura aprovada

## Veto Conditions

| Condicao | Acao |
|----------|------|
| Estrutura contradiz PRD | VETO — debater: ajustar estrutura ou ajustar PRD |
| Avancar sem ciclo fractal | VETO — co-desenvolver -> estruturar -> refinar -> validar |
| Usuario nao aprovou | VETO — nao avancar sem "fechou" |
| Modulos/fases sem objetivo claro | VETO — cada fase precisa de objetivo definido |

---

## Error Handling

| Cenario | Acao |
|---------|------|
| Programa muito longo (muitas fases) | Decompor em blocos. Priorizar. Debater com usuario |
| Overlap entre fases/modulos | Debater com usuario. Consolidar ou separar com clareza |
| Expert quer pular estruturacao | Nao pode — PRD exige. Explicar que sem estrutura, sessoes ficam soltas |
| Sequencia nao faz sentido | Reordenar. Apresentar alternativa. Usuario decide |
| Modalidade nao clara afeta estrutura | Voltar pra PRD. Decidir modalidade primeiro |
